# 통변역 툴 (Meet Interpreter)

구글밋 인터뷰용 실시간 통역 로컬 웹앱.

- **인터뷰어(한국어/영어)** 가 말하면 → **중국어 음성**으로 통역해서 인터뷰이에게 들려줌 (밋 마이크로 주입)
- **인터뷰이(중국어)** 가 대답하면 → 소리 없이 **한국어 + 영어 텍스트**로 화면에만 표시

```
[인터뷰어 실제 마이크] ─(한/영)→ 브라우저 → 서버 → Google STT
   → Translate(→중국어) → TTS → [BlackHole] → 구글밋 마이크 → 인터뷰이가 들음

[구글밋 탭 오디오] ─(인터뷰이 중국어)→ 브라우저 → 서버 → Google STT(중국어)
   → Translate(→한/영) → 화면 텍스트만 (소리 없음)
```

---

## 1. 사전 준비

### (a) Node.js
Node 18 이상. 확인: `node -v`

### (b) Google Cloud 설정
1. [Google Cloud Console](https://console.cloud.google.com)에서 프로젝트 생성(또는 기존 프로젝트).
2. 아래 3개 API 사용 설정(Enable):
   - **Cloud Speech-to-Text API**
   - **Cloud Translation API**
   - **Cloud Text-to-Speech API**
3. **서비스 계정** 생성 → 키(JSON) 발급 → 다운로드한 파일을 이 폴더에 `gcp-key.json` 으로 저장.
   - IAM 역할은 위 3개 서비스 사용 권한이면 됨(간단히는 `Editor` 도 가능하나 최소권한 권장).
4. 결제(Billing) 활성화 필요(무료 크레딧/무료 티어 있음).

### (c) BlackHole (가상 오디오 케이블, macOS)
중국어 TTS 를 구글밋 마이크로 넣기 위해 필요합니다.

```bash
brew install blackhole-2ch
```
설치 후 로그아웃/재로그인 또는 재부팅하면 오디오 장치 목록에 **BlackHole 2ch** 가 나타납니다.
(Homebrew 가 없으면 https://existential.audio/blackhole/ 에서 설치)

---

## 2. 설치 & 실행

```bash
cd ~/Documents/meet-interpreter
cp .env.example .env      # 필요시 값 수정
npm install
npm start
```

브라우저(Chrome 권장)에서 **http://localhost:3000** 접속.

> ⚠️ 반드시 `localhost` 로 접속하세요. 마이크/탭 캡처/오디오 라우팅은 보안 컨텍스트(localhost)에서만 동작합니다.

---

## 3. 오디오 라우팅 세팅 (핵심)

인터뷰 시작 전에 한 번만 맞춰두면 됩니다.

### 구글밋 쪽
- 구글밋 설정 → **마이크** 를 **BlackHole 2ch** 로 선택.
  → 이렇게 하면 인터뷰이는 "우리 앱이 만든 중국어 TTS"만 듣게 됩니다.
- 구글밋 **스피커** 는 본인 **헤드폰**으로 선택.

### 이 앱 쪽 (localhost:3000 화면)
- **🎙️ 인터뷰어 마이크**: 본인이 말할 실제 마이크(내장/USB 등) 선택.
- **🔊 중국어 TTS 출력 장치**: **BlackHole 2ch** 선택. (자동 감지되면 이미 선택됨)

### 인터뷰어(본인)도 중국어 음성을 듣고 싶다면
- 화면의 **"중국어 음성 여기서도 듣기(모니터)"** 체크박스를 켜세요. 기본 스피커/헤드폰으로도 같이 재생됩니다.

> 💡 **헤드폰 필수**: 스피커로 인터뷰이 목소리를 틀면 실제 마이크에 다시 잡혀 인식이 꼬입니다. 헤드폰을 쓰세요.

---

## 4. 인터뷰 진행

1. 구글밋 회의 입장(마이크 = BlackHole, 스피커 = 헤드폰).
2. localhost:3000 에서 **"▶ 인터뷰어 마이크 시작"** 클릭.
3. **"▶ 인터뷰이 캡처 시작 (밋 탭 공유)"** 클릭 →
   공유 창에서 **구글밋 탭**을 고르고 **"탭 오디오도 공유"** 를 반드시 켠 뒤 공유.
4. 이제:
   - 한국어/영어로 말하면 → 오른쪽 왼쪽 패널(인터뷰어)에 원문+중국어가 뜨고, 중국어 음성이 밋으로 나갑니다.
   - 인터뷰이가 중국어로 답하면 → 인터뷰이 패널에 한국어/영어 번역 텍스트가 뜹니다(소리 없음).

---

## 5. 자주 겪는 문제

| 증상 | 원인 / 해결 |
|---|---|
| 인터뷰이가 내 목소리(한국어)를 그대로 들음 | 구글밋 마이크가 BlackHole 이 아님. 밋 설정에서 마이크를 BlackHole 2ch 로 변경. |
| 중국어 음성이 인터뷰이에게 안 들림 | 앱의 TTS 출력 장치가 BlackHole 인지 확인. |
| "탭 오디오가 공유되지 않았습니다" | 화면 공유 시 **"탭 오디오도 공유"** 체크를 켜야 함. 창(Window)이 아니라 **탭(Tab)** 을 공유해야 오디오가 잡힘. |
| 인식이 자꾸 끊기거나 이상함 | 헤드폰 사용 확인(스피커 에코 방지). 조용한 환경 권장. |
| 출력 장치 목록에 BlackHole 이 없음 | BlackHole 설치 후 재부팅. `brew list` 로 설치 확인. |
| TTS 목소리/속도 변경 | `.env` 의 `TTS_VOICE`, `TTS_RATE` 수정 후 재시작. |

---

## 6. 비용 참고 (대략)
- Speech-to-Text: 스트리밍 분당 과금(무료 60분/월 티어 있음).
- Translation: 문자 수 과금(월 50만 자 무료 티어).
- Text-to-Speech: WaveNet 문자 수 과금(월 100만 자 무료 티어).

인터뷰 1시간 기준 소액 수준이지만, 정확한 요금은 GCP 콘솔에서 확인하세요.

## 언어 코드 (server.js 상단 `LANG` 에서 변경 가능)
- 인터뷰어 인식: `ko-KR` + 대체 `en-US`
- 인터뷰이 인식: `cmn-Hans-CN` (표준 중국어/간체)
- 번역 타겟: 중국어 `zh-CN`, 한국어 `ko`, 영어 `en`
- 중국어 TTS: `cmn-CN` / 음성 `cmn-CN-Wavenet-A`
